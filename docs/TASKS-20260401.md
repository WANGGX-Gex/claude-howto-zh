### 支柱 4：新手引导

- [ ] **T-001** — 编写 `scripts/quickstart.sh`
  - 检测 `~/.claude/` 目录（如不存在则经用户确认后创建）
  - 将第一个斜杠命令 + CLAUDE.md + 技能复制到用户的配置中

  - 完成时打印下一步操作

- [ ] **T-002** — 创建 `QUICKSTART.md`（前 15 分钟可视化指南）
  - 以代码块形式展示带注释的终端步骤（尽可能避免 PNG 截图；优先使用 ASCII 或 Mermaid）
  - 记录哪些命令产生每个视觉效果，以便可以重新截取

- [ ] **T-003** — 为所有 10 个模块 README 添加难度徽章
  - 格式：`> 🟢 **Beginner** | ⏱ 30 minutes`，放在每个模块 README 顶部
  - 在徽章下方添加"你将构建什么"预览

### 支柱 7：活的课程体系

- [ ] **T-004** — 创建 `WHATS-NEW.md`
  - 格式：`## CC vX.X — DATE`，每个功能变更一个条目
  - 至少为当前 CC 版本添加一个条目

- [ ] **T-005** — 为所有 10 个模块 README 添加版本徽章
  - 格式：`> ✅ Verified against Claude Code **vX.X.X** · Last verified: YYYY-MM-DD`
  - 在每个模块的 frontmatter 中添加 `cc_version_verified`

- [ ] **T-006** — 创建 `.github/workflows/staleness-check.yml`
  - 调度：每周一 09:00 UTC
  - 从 frontmatter 中读取每个模块的 `last_verified` 日期（使用 `yq` 或 10 行 Python 脚本）
  - 为任何超过 30 天未验证的模块创建 GitHub Issue
  - 如已存在相同标题的 open issue 则跳过

### 支柱 6：内容质量（CLAUDE.md）

### 支柱 2：AI 代理索引（生成器）

  - 输出 `AGENT-INDEX.md`（人类可读的摘要）
