# 贡献者行为准则

## 我们的承诺

我们致力于为所有人提供一个友好且鼓舞人心的社区。我们承诺，参与社区的每个人都能获得无骚扰的体验，无论年龄、体型、种姓、残疾、种族、性别特征、性别认同和表达、经验水平、教育背景、社会经济地位、国籍、外貌、种族、宗教或性取向。

我们致力于提供一个积极、包容和安全的环境，让所有贡献者都能感到被尊重、被重视，并有能力做出最好的工作。

## 我们的标准

有助于营造积极环境的行为示例包括：

### 尊重他人
- 使用友善和包容的语言
- 尊重不同的观点、看法和经验
- 优雅地接受建设性批评
- 尊重文化和语言差异
- 承认并庆祝多元背景

### 协作精神
- 共同努力解决冲突
- 给予认可并感谢他人的贡献
- 帮助他人学习和成长
- 分享知识并提问
- 支持社区成员

### 专业态度
- 保持讨论的专注和有效性
- 避免贬低或侮辱性言论
- 在 Issue 和讨论中保持主题
- 使用清晰和尊重的沟通方式
- 假设他人的消息出于善意

### 包容开放
- 欢迎新的贡献者
- 帮助项目新成员入门
- 提供建设性反馈
- 包容不同观点的人
- 为安静的声音创造表达空间

## 不可接受的行为

以下行为被视为骚扰，在我们的社区中是不可接受的：

### 骚扰和歧视
- 关于性别、性别认同和表达、性取向、残疾、精神疾病、神经多样性、外貌、体型、年龄、种族或宗教的冒犯性评论
- 对他人生活方式选择和做法的不受欢迎的评论
- 故意使用错误的性别称谓或使用他人已放弃的名字
- 无端或偏离主题的色情图片或行为
- 未经同意或在对方要求停止后的身体接触和模拟身体接触

### 滥用行为
- 针对他人的暴力威胁或暴力语言
- 煽动暴力或针对任何个人的暴力语言
- 蓄意恐吓
- 跟踪或尾随
- 通过反复联系进行骚扰
- 其他可合理视为不当的行为

### 在线骚扰
- 持续干扰在线讨论
- 公布他人的私人信息（人肉搜索）
- 不受欢迎的性关注或挑逗
- 恶意挑衅或故意制造混乱
- 垃圾信息或重复广告
- 散布虚假或误导信息

### 报复行为
- 对举报违规行为的人采取报复行动
- 因某人举报了不可接受的行为而不公平地对待他们
- 威胁以举报作为报复手段

## 适用范围

本行为准则适用于：
- 所有社区空间（GitHub Issue、Pull Request、讨论等）
- 项目仓库和文档
- 由项目组织或代表项目的活动和聚会
- 个人代表项目或社区出现的公共场所
- 与项目事务相关的私人通信

本行为准则适用于所有项目空间，也适用于个人在公共场所代表项目或社区的情况。代表项目或社区的示例包括使用官方项目电子邮件地址、通过官方社交媒体账号发帖，或在线上或线下活动中担任指定代表。

## 执行

社区领导者负责阐明和执行可接受行为标准，并将对任何他们认为不当、具有威胁性、冒犯性或有害的行为采取适当和公正的纠正措施。

### 举报违规行为

如果你经历或目睹了不可接受的行为，或有任何其他疑虑，请通过以下方式举报：

1. **电子邮件**: 联系项目维护者（如有可用的联系方式）
2. **GitHub Issue**: 私密举报功能（如果在你的 GitHub 设置中可用）
3. **私信**: 直接联系项目维护者

请提供：
- 你的联系方式
- 涉及人员的姓名（如果你觉得方便分享的话）
- 事件描述
- 上下文和任何相关链接
- 事件对你的影响

**所有举报将被保密处理。**

### 调查和回应

社区领导者将：
- 及时公正地审查和调查所有投诉
- 对举报者保密
- 就举报状态回复举报者
- 根据调查结果采取适当行动

可能的行动包括：
- 私下警告或指导
- 公开道歉（如果违规者同意）
- 临时或永久移出社区空间
- 取消贡献者权限
- 根据需要向 GitHub 或其他平台升级

### 申诉流程

如果你认为某项决定有误，你可以通过以下方式申诉：
1. 提供新的信息或上下文
2. 请求多位维护者进行审查
3. 在初始决定后 30 天内跟进

## 尊重的分歧指南

在技术社区中意见分歧是自然的。以下是尊重地表达分歧的方式：

### 推荐做法
- 关注想法，而非人身
- 提出澄清性问题
- 承认对方观点中的有效之处
- 使用"我"陈述句（"我不同意，因为..."）
- 引用来源和证据
- 感谢他人的观点

### 避免做法
- 进行人身攻击
- 使用轻蔑的语言（"这太蠢了"）
- 假设对方出于恶意
- 翻旧账，提及不相关的过去冲突
- 升级语气或变得敌意
- 故意歪曲他人的观点

### 尊重分歧的示例
```
不推荐: "这是个蠢主意，你显然不了解这个项目。"

推荐: "我理解你的观点，但我担心这种方法在大型项目中
可能无法很好地扩展。你是否考虑过这个基准测试
中展示的性能影响？我很想听听你对此的看法。"
```

## 搭建桥梁

我们相信对话和和解的力量：

- 如果有人指出你的行为问题，请倾听并反思
- 如果你造成了伤害，愿意道歉
- 支持他人的学习和成长
- 认识到人们会犯错
- 关注解决方案，而非责备
- 假设善意的同时让人承担责任

## 遵循本准则的好处

通过维护一个尊重和包容的社区，我们：
- 吸引拥有不同视角的多元贡献者
- 通过协作创造更好的解决方案
- 建立更强的关系
- 使项目对新人更加友好
- 产出更高质量的文档和示例
- 培养学习和成长的文化

## 特别注意事项

### 对维护者
- 以身作则，遵守本准则
- 对举报及时回应
- 在决策方面保持透明
- 帮助公正解决冲突
- 为所有人创造安全的环境

### 对贡献者
- 假设维护者和其他贡献者出于善意
- 提供建设性反馈
- 支持新人
- 帮助维护社区标准
- 对社区文化负责

### 对所有人
- 记住每条消息背后都是一个真实的人
- 如果感到沮丧，休息一下
- 如果需要支持，联系社区领导者
- 帮助创建你想参与的社区
